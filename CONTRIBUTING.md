# Contributing to zhilicon-developer-docs

Documentation contributions are among the most valuable ways to help the Zhilicon developer community.

## Ways to Contribute

- **Fix errors** — incorrect API signatures, outdated instructions, broken links
- **Improve clarity** — better explanations, more examples, clearer diagrams
- **Add guides** — new integration tutorials, framework bridge docs
- **Translate** — documentation in languages other than English (open an issue first)

## Content Guidelines

- Write in plain, direct English — developers skimming for answers are the audience
- Use code examples liberally — one working snippet beats a paragraph of prose
- Every code example must be tested; mark untested examples with `# Untested`
- Do not document internal implementation details or undocumented APIs

## DCO Sign-Off

All contributions require a Developer Certificate of Origin sign-off:
```
git commit -s -m "docs: fix API signature for zhilicon.Device.allocate()"
```

## PR Guidelines

- One topic per PR
- Link to the live page on developers.zhilicon.ai if fixing an existing page
- Screenshots welcome for UI or workflow changes

## Review Process

Reviewed by `@zhilicon-ai/docs`. For API reference changes, relevant engineering teams co-review for accuracy.
